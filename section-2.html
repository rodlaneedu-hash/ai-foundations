How Generative AI Works

Purpose: Build a conceptual mental model — just enough to understand why it does what it does (and doesn't do what people assume it does). Misconceptions are dissolved as each concept lands.

Six building blocks, each paired with the misconception it dissolves:

1. Training data — patterns

Generative AI is trained on massive amounts of text (books, websites, articles, code). During training, it learns patterns — which words and ideas tend to appear together, in what order, in what situations.

Misconception dissolved

"AI has been programmed with knowledge/facts." It hasn't. It has patterns. No one sat down and taught it that Wellington is the capital of New Zealand — it encountered that pattern frequently enough in training data to reproduce it.

2. Token prediction

When you give it a prompt, it generates a response one word at a time (technically called a "token"), each time asking: "given everything so far, what word should come next?" It does this thousands of times to produce a response.

Misconception dissolved

"AI understands what it's saying." It doesn't. It's really good at predicting what comes next based on patterns. The output often reads as if it understands, because human language follows patterns, and the model is very good at reproducing them. But there is no comprehension behind it.

3. It doesn't give the same answer twice

Because there's a degree of randomness in how it picks the next word, the same prompt can produce different outputs each time. It's not retrieving a stored answer — it's generating a new one every time.

Misconception dissolved

"There's a 'right answer' in there if I ask the right way." There isn't. Every response is freshly generated. This is also why two people can get different answers to the same question.

4. No memory, no knowledge, no reasoning

It has no access to the internet (unless given tools). It has no memory between conversations (unless designed to). It cannot verify its own claims. It doesn't actually think things through the way we do — it produces text that looks like reasoning because when humans reason, we use patterns of language, and the AI is very good at copying those patterns. Newer models are getting better at multi-step reasoning tasks, but under the hood, it's still predicting patterns, not actually understanding anything.

Misconception dissolved

"AI knows things / AI is always right / AI can fact-check itself." It can't. It will state a falsehood with the same confidence as a fact, because sounding confident is just another pattern it learned. It sounds confident whether it's right or wrong. This is called hallucination — and it's getting better with each new version, but it won't go away. It's inherent to how the technology works.

5. It's not a search engine

When you type something into Google, it searches an index of web pages and shows you links to information that already exists. When you type something into a generative AI tool, the underlying language model is not usually going out to the internet to look things up in that way. Instead, it generates a response one word at a time, based on patterns it has learned from very large amounts of text. In other words, it is constructing an answer that sounds right, rather than retrieving a specific page it has stored.

This means the model itself cannot reliably tell you "I got this from here," because it does not keep a neat list of sources it can point back to. If you ask a basic AI chatbot for references, it may sometimes invent ones that look real — with convincing authors, titles, and journals — because it has learned what references usually "look" like, not because it has actually checked a real database. Newer tools, like Perplexity and some versions of ChatGPT, add a separate search step on top of the model so they can show you real web pages and citations, but underneath they are still using a language model that generates text rather than a search engine that simply finds documents.

Misconception dissolved

"I asked AI, and it said..." is not the same as "I looked it up and found..." Treating AI output as a retrieved fact is the single fastest way to get burned. If you need a fact, go to the source. Use the AI to help you think, draft, and structure — not to tell you what's true.

6. Its knowledge has a use-by date

Every AI model was trained on data up to a specific point in time — its knowledge cutoff. It doesn't know what happened after that date. If you ask about recent events, new legislation, current statistics, or anything that's changed since its training ended, it will either tell you it doesn't know or — more dangerously — generate a plausible-sounding answer based on older patterns. It has no way of knowing that its information is outdated.

Misconception dissolved

"AI has the latest information." It doesn't, unless the tool has been specifically connected to the internet or given access to current data. Even then, the core model's knowledge has a fixed boundary. Always check whether the information you're asking about falls within or beyond that boundary.

Key Landing Point

Generative AI works by predicting patterns. That's all it does. It is extraordinarily good at producing fluent, plausible, useful text. But it does not know, understand, reason, or verify. Every strength and every risk flows from this single fact. Which is why one principle applies every time you use it: a human must always be in the loop — checking, interpreting, and taking responsibility for the output. Not sometimes. Every time.
